# Local Transcription CLI

Небольшой CLI-скрипт для локальной транскрибации аудио и видео через `ffmpeg` и `gigaam-mlx`.

Скрипт:
- принимает аудио или видеофайл;
- при необходимости конвертирует его в `wav 16kHz mono`;
- разбивает запись на сегменты;
- прогоняет каждый сегмент через `gigaam-mlx`;
- сохраняет результат в `.txt` с таймкодами.

## Зачем это нужно

Это простой персональный инструмент для быстрой офлайн-транскрибации интервью, звонков, созвонов и заметок без облачных сервисов.

## Возможности

- Поддержка `mp4`, `mov`, `avi`, `mkv`, `webm`, `m4a`, `mp3`, `wav`, `flac`, `aac`, `ogg`
- Модели `rnnt` и `ctc`
- Настраиваемая длина сегмента
- Сохранение результата в текстовый файл
- Минимальная постобработка шума из вывода распознавания

## Требования

- Python 3.10+
- `ffmpeg`
- `gigaam-mlx`

## Установка

### 1. Клонировать репозиторий

```bash
git clone https://github.com/JoehFlu/local-transcription-cli.git
cd local-transcription-cli
```

### 2. Подготовить окружение Python

```bash
python3 -m venv .venv
source .venv/bin/activate
```

Если у вас уже есть отдельное окружение под `gigaam-mlx`, можно использовать его.

### 3. Установить `ffmpeg`

На macOS с Homebrew:

```bash
brew install ffmpeg
```

Проверка:

```bash
ffmpeg -version
```

### 4. Установить и проверить `gigaam-mlx`

Установку лучше выполнять по актуальной инструкции проекта `gigaam-mlx`.

После установки убедитесь, что команда доступна:

```bash
gigaam-mlx --help
```

## Использование

Быстрый старт для текущего локального окружения:

```bash
source ~/gigaam-env/bin/activate
python3 transcribe.py audio.mp4 --model rnnt --segment 20
```

Базовый запуск:

```bash
python3 transcribe.py video.mp4
```

Выбор модели и длины сегмента:

```bash
python3 transcribe.py video.mp4 --model rnnt --segment 20
```

Явно указать путь к выходному файлу:

```bash
python3 transcribe.py audio.mp3 --output result.txt
```

## Пример результата

```text
ТРАНСКРИПЦИЯ
Модель: RNNT | Сегмент: 20 сек
======================================================================

[0:00:00]
Пример распознанного текста...
```

## Как это работает

1. Входной файл проверяется на существование и поддерживаемый формат.
2. Если нужно, выполняется конвертация в `wav 16kHz mono`.
3. Аудио режется на сегменты фиксированной длины.
4. Каждый сегмент отправляется в `gigaam-mlx`.
5. Результат очищается от служебного мусора и сохраняется в `.txt`.

## Ограничения

- Таймкоды сейчас считаются по размеру сегмента, а не по фактической длительности каждого куска.
- Постобработка текста остаётся минимальной и может требовать ручной правки.
- Для очень длинных файлов производительность зависит от скорости `ffmpeg` и модели распознавания.

## Roadmap

- Эксперименты с локальной постобработкой через Ollama
- Режим суммаризации или очистки текста через локальную LLM, например `qwen3.5:4b`

## Идея для следующего этапа

- `LLM post-processing`: очистка, нормализация, саммари, выделение тезисов

Тогда `gigaam-mlx` будет отвечать за распознавание речи, а Ollama с локальной моделью вроде `qwen3.5:4b` сможет:

- убирать слова-паразиты;
- исправлять пунктуацию;
- делать краткое резюме разговора;
- извлекать action items;
- переводить устную речь в более читаемый письменный текст.
